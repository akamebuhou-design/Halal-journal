① Setup Guide
② Sign In
Back up your trades in your own Google Drive — not ours. We never see your data. One-time OAuth setup below.
1
Create a Google Cloud Project
Open console.cloud.google.com/projectcreate, name the project "Halal Journal" and click Create.
2
Enable the Drive API
In the left sidebar go to APIs & Services → Library. Search for and click Enable.
3
Configure the OAuth Consent Screen
Go to APIs & Services → OAuth consent screen → choose External → fill in any App name → under Scopes add the scope:
What is drive.file? This is the most privacy-preserving Drive scope available. It grants access only to files this app creates — your photos, documents, and all other Drive files remain completely invisible to the app.
Finally, add your Gmail address as a Test User and save.
4
Create an OAuth Client ID
Go to Credentials → + Create Credentials → OAuth client ID → select Web Application.

Under Authorised JavaScript origins, add the exact URL where you're running this file — for example: Click Create.
5
Copy your Client ID
In the popup (or the Credentials list) you'll see a Client ID ending in Copy it, then click Next below to paste it in Tab ②.
🔒
Security Guarantee
Zero-knowledge · drive.file scope only
✓
The app uses the drive.file scope — it can only read or write the single JSON file it creates. Every other file in your Drive is cryptographically inaccessible.
✓
Your OAuth access token is stored in memory only and is never written to localStorage, cookies, or any server.
✓
All Drive API calls go directly from your browser to Google's servers. No data passes through any third-party infrastructure.
✓
You can revoke access at any time from myaccount.google.com/permissions.
Paste the Client ID from Step 5 of the guide. It is saved in your browser's localStorage and never transmitted to any server. No sync happens until you click Sign in with Google.
